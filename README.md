# AIClient2API 用户插件集合



本仓库收录了 [AIClient2API](https://github.com/justlovemaki/AIClient2API) 的用户自制插件。

## 如何使用插件市场

在 AIClient2API 管理后台 → 插件管理页面，将以下地址填入「远程 market.json 地址」输入框，点击加载即可浏览和安装本仓库的所有插件：

```
https://raw.githubusercontent.com/laobiao651-commits/aiclient2api-plugins/main/market.json
```

## 插件列表

### proxy-binding — 号池代理绑定

> 当前版本：**v1.0.3** | 最低系统版本：3.1.1

为号池中每个账号独立绑定出站代理 IP，支持代理池管理、批量绑定、IP 脱敏显示、节点联动筛选等功能。

#### 功能特性

- **账号独立代理**：为每个号池账号单独指定出站代理，互不干扰
- **代理池管理**：统一维护代理节点，支持单个添加和批量导入
- **批量操作**：多选账号后一键批量分发代理或批量解绑
- **节点联动筛选**：点击节点池中的某个节点，账号列表自动筛选出绑定该节点的账号
- **IP 脱敏显示**：代理 URL 默认脱敏（密码替换为 `****`，IP 中间两段替换为 `***.***`），可一键切换全显
- **账号标识脱敏**：账号名默认显示为 `lax***com` 格式，可一键切换全显
- **凭据文件列**：账号列表显示对应的凭据文件名称
- **使用统计**：显示每个账号的调用次数、错误次数和最后使用时间
- **分页 + 虚拟滚动**：支持大规模账号场景，分页条数支持自定义输入
- **加载状态**：数据加载中显示动画，无数据时显示友好提示
- **深色 / 浅色主题**：支持一键切换，偏好持久化保存
- **导入 / 导出**：支持代理配置的 JSON 导入导出
- **移动端适配**：账号列表在移动端自动切换为卡片式布局

#### 安装方法

**方式一：通过插件市场安装（推荐）**

1. 打开 AIClient2API 管理后台 → 插件管理
2. 在「远程 market.json 地址」输入框填入：
   ```
   https://raw.githubusercontent.com/laobiao651-commits/aiclient2api-plugins/main/market.json
   ```
3. 点击「加载」，找到 `proxy-binding` 插件，点击「安装」
4. 安装完成后重启服务，插件自动生效

**方式二：手动安装**

1. 下载最新版 zip：
   ```
   https://github.com/laobiao651-commits/aiclient2api-plugins/releases/latest/download/proxy-binding.zip
   ```
2. 解压，将 `proxy-binding/` 目录放入项目的 `src/plugins-user/` 目录下
3. 重启服务

---

#### 更新方法

**方式一：通过插件市场更新（推荐）**

1. 打开管理后台 → 插件管理
2. 如果 `proxy-binding` 有新版本，插件卡片会显示「有新版本」提示
3. 点击「更新」，系统自动下载并替换，重启服务生效

**方式二：手动更新**

> ⚠️ **重要**：zip 包中的 `proxy-pool.json` 是空数据，手动更新前务必先备份，否则已配置的代理节点数据会丢失。通过插件市场更新不受此影响。

1. 备份代理池数据：
   ```bash
   cp src/plugins-user/proxy-binding/proxy-pool.json ~/proxy-pool.backup.json
   ```
2. 下载新版 zip 并解压，将 `proxy-binding/` 目录覆盖到 `src/plugins-user/proxy-binding/`
3. 还原代理池数据：
   ```bash
   cp ~/proxy-pool.backup.json src/plugins-user/proxy-binding/proxy-pool.json
   ```
4. 重启服务

#### 使用说明

**首次使用**

1. 安装并重启服务后，在管理后台插件列表找到 `proxy-binding`，点击描述中的「打开管理界面」链接
2. 在页面顶部 API Key 输入框填入你的 API Key（即配置文件中 `REQUIRED_API_KEY` 的值），点击「连接」
3. 在左侧「节点池」区域点击「+ 添加」，填入代理节点名称和地址
4. 在右侧「账号列表」中选中账号，点击「指派」，从节点池中选择代理节点，点击「立即生效」

**批量操作**

- 勾选多个账号后，顶部出现批量操作栏
- 「批量分发」：为所有选中账号统一指派同一个代理节点
- 「全解绑」：批量移除选中账号的代理绑定

**节点联动筛选**

- 点击左侧节点池中的某个节点卡片，右侧账号列表自动筛选出绑定了该节点的账号
- 再次点击同一节点卡片取消筛选，恢复显示全部账号

**脱敏 / 全显切换**

- 节点池右上角「显示 / 隐藏」按钮：控制节点 URL 的脱敏显示
- 账号列表筛选栏右侧「账号全显 / 账号脱敏」按钮：控制账号标识的脱敏显示

#### 代理地址格式

| 协议 | 格式示例 |
|------|---------|
| SOCKS5（带认证） | `socks5://user:pass@1.2.3.4:1080` |
| SOCKS5（无认证） | `socks5://1.2.3.4:1080` |
| HTTP（带认证） | `http://user:pass@1.2.3.4:8080` |
| HTTP（无认证） | `http://1.2.3.4:8080` |

#### 批量导入节点格式

在「≡ 批量」弹窗中，每行一个节点，支持两种格式：

```
节点名称|代理地址
美国节点|socks5://user:pass@1.2.3.4:1080
日本节点|http://user:pass@5.6.7.8:8080
socks5://9.10.11.12:1080
```

名称和地址用 `|` 分隔；只填地址不填名称时，系统自动生成名称。
